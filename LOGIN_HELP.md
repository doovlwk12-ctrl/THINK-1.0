# مساعدة تسجيل الدخول

## إذا ظهرت رسالة "البريد الإلكتروني أو كلمة المرور غير صحيحة"

تظهر هذه الرسالة عندما لا يوجد مستخدم في قاعدة البيانات بالبريد وكلمة المرور المدخلة، أو عندما تكون قاعدة البيانات غير مهيأة.

### 1. تنفيذ أوامر Prisma بشكل صحيح

استخدم الأمر **بدون نقطة** في النهاية:

```bash
npx prisma generate
```

(وليس `prisma generate.`)

### 2. تهيئة قاعدة البيانات وإنشاء الجداول

```bash
npx prisma db push
```

أو إذا كنت تستخدم migrations:

```bash
npx prisma migrate dev
```

### 3. إنشاء المستخدمين التجريبيين

```bash
npm run db:seed
```

### 4. استخدام الحسابات التجريبية

بعد تشغيل الـ seed، استخدم أحد الحسابات التالية:

| الدور   | البريد الإلكتروني     | كلمة المرور   |
|--------|------------------------|----------------|
| عميل   | `client@test.com`      | `password123`  |
| مهندس  | `engineer@test.com`     | `password123`  |
| مدير   | `admin@test.com`        | `password123`  |

### 5. التأكد من ملف .env

تأكد من وجود ملف `.env` في جذر المشروع ويحتوي على:

- `DATABASE_URL` — رابط قاعدة البيانات (مثل `file:./dev.db` لـ SQLite أو رابط PostgreSQL)
- `NEXTAUTH_SECRET` — أي نص عشوائي (32 حرفاً على الأقل)
- `NEXTAUTH_URL="http://localhost:3000"` (أو الرابط الذي تشغّل عليه التطبيق)

بعد تنفيذ الخطوات 1–4 واستخدام البريد وكلمة المرور أعلاه، يجب أن يعمل تسجيل الدخول.
