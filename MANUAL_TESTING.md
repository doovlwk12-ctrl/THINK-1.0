# قائمة الاختبار اليدوي — منصة فكرة

استخدم هذه القائمة للتحقق من المنصة يدوياً قبل إصدار أو بعد تعديلات.  
ضع ✓ في المربع عند إكمال كل بند.

**الحسابات التجريبية:** راجع [README.md](./README.md#-بيانات-تسجيل-تجريبية)  
`client@test.com` / `engineer@test.com` / `admin@test.com` — كلمة المرور: `password123`

---

## 1. التشغيل والبيئة

- [ ] تشغيل `npm run dev` بنجاح
- [ ] فتح `http://localhost:3000` — الصفحة تُحمّل
- [ ] لا تظهر أخطاء حمراء في كونسول المتصفح (F12)

---

## 2. الصفحة الرئيسية

- [ ] الصفحة الرئيسية تُعرض بشكل صحيح
- [ ] قسم "باقات الخدمة" أو الباقات يظهر
- [ ] الروابط في الهيدر تعمل (تسجيل الدخول، إنشاء حساب، إلخ)
- [ ] الوضع الليلي يعمل إن وُجد (تبديل إن وُجد)

---

## 3. التسجيل وإنشاء حساب

- [ ] فتح `/register` — صفحة التسجيل تظهر
- [ ] تعبئة الاسم، البريد، كلمة المرور، الهاتف (إن وُجد)
- [ ] إرسال النموذج — يتم إنشاء الحساب أو تظهر رسالة مناسبة
- [ ] إعادة التوجيه إلى لوحة التحكم أو صفحة تسجيل الدخول

---

## 4. تسجيل الدخول

- [ ] فتح `/login` — صفحة تسجيل الدخول تظهر
- [ ] إدخال بريد وكلمة مرور خاطئة — تظهر رسالة خطأ مناسبة
- [ ] تسجيل الدخول كـ **عميل:** `client@test.com` / `password123`
  - [ ] يتم التوجيه إلى `/dashboard` (لوحة العميل)
  - [ ] اسم المستخدم أو القائمة تظهر في الهيدر
- [ ] تسجيل الخروج ثم الدخول كـ **مهندس:** `engineer@test.com` / `password123`
  - [ ] يتم التوجيه إلى `/engineer/dashboard`
- [ ] تسجيل الخروج ثم الدخول كـ **مدير:** `admin@test.com` / `password123`
  - [ ] يتم التوجيه إلى `/admin/dashboard`

---

## 5. لوحة تحكم العميل

(تسجيل الدخول كـ `client@test.com`)

- [ ] لوحة التحكم تُعرض (`/dashboard`)
- [ ] قائمة الطلبات أو رسالة "لا توجد طلبات" تظهر
- [ ] وجود زر أو رابط "إنشاء طلب جديد" أو ما شابه
- [ ] فتح صفحة الملف الشخصي إن وُجدت (`/dashboard/profile`) — تعمل بدون خطأ

---

## 6. إنشاء طلب (عميل)

- [ ] الدخول إلى إنشاء طلب (من الرئيسية أو من لوحة التحكم)
- [ ] اختيار باقة — القائمة أو الباقات تظهر
- [ ] اختيار باقة والمتابعة — الانتقال للخطوة التالية
- [ ] تعبئة بيانات الطلب (العنوان، المساحة، التفاصيل، إلخ حسب النموذج)
- [ ] إرسال الطلب — تظهر رسالة نجاح أو يتم التوجيه لصفحة الطلب
- [ ] الطلب الجديد يظهر في لوحة تحكم العميل

---

## 7. صفحة طلب واحد (عميل)

- [ ] من لوحة التحكم، فتح أحد الطلبات (`/orders/[id]`)
- [ ] تفاصيل الطلب تظهر (الحالة، الباقة، التاريخ، إلخ)
- [ ] أزرار الإجراءات تظهر حسب الحالة (دفع، محادثة، تعديلات، إلخ)
- [ ] لا أخطاء في الكونسول

---

## 8. الدفع (إن وُجد طلب بانتظار الدفع)

- [ ] فتح صفحة الدفع للطلب (`/orders/[id]/payment` أو من داخل صفحة الطلب)
- [ ] اختيار طريقة الدفع أو إكمال المحاكي
- [ ] إتمام الدفع — تتحدث حالة الطلب أو تظهر رسالة نجاح

---

## 9. المحادثة (عميل ↔ مهندس)

- [ ] من صفحة الطلب، فتح المحادثة (تبويب أو رابط المحادثة)
- [ ] مربع إدخال الرسالة يظهر
- [ ] إرسال رسالة — تظهر في المحادثة دون خطأ
- [ ] (اختياري) تسجيل الدخول كمهندس وفتح نفس الطلب — الرسائل تظهر

---

## 10. رفع المخططات وإرسالها (مهندس)

(تسجيل الدخول كـ `engineer@test.com`)

- [ ] لوحة تحكم المهندس تُعرض (`/engineer/dashboard`)
- [ ] قائمة الطلبات (المُعيَّنة أو المتاحة) تظهر
- [ ] فتح طلب — صفحة الطلب تظهر
- [ ] رفع مخطط (PDF أو صورة) — يتم الرفع دون خطأ
- [ ] إرسال المخطط للعميل — تظهر رسالة نجاح أو تحديث في الواجهة

---

## 11. التعديلات والدبابيس (إن وُجدت)

- [ ] من صفحة الطلب (عميل أو مهندس)، فتح قسم التعديلات/الدبابيس
- [ ] إضافة نقطة تعديل أو عرض التعديلات — الواجهة تعمل دون خطأ
- [ ] (عميل) شراء تعديلات إضافية إن وُجد — الصفحة أو النموذج يعمل

---

## 12. تمديد الموعد و إكمال الطلب

- [ ] (مهندس) طلب تمديد الموعد إن وُجد — يعمل دون خطأ
- [ ] (عميل) تأكيد إكمال الطلب إن وُجد — يعمل دون خطأ

---

## 13. لوحة الإدارة

(تسجيل الدخول كـ `admin@test.com`)

- [ ] لوحة الإدارة تُعرض (`/admin/dashboard`)
- [ ] الإحصائيات أو ملخص الطلبات يظهر
- [ ] قائمة الطلبات (`/admin/orders`) — تظهر وتُفتح دون خطأ
- [ ] إدارة الباقات (`/admin/packages`) — عرض الباقات، إضافة/تعديل إن وُجد
- [ ] إدارة المهندسين (`/admin/engineers`) — القائمة أو الطلبات تظهر

---

## 14. عام

- [ ] التنقل بين الصفحات لا يسبب أخطاء في الكونسول
- [ ] الوضع الليلي يعمل في الصفحات المختبرة (إن وُجد)
- [ ] الهيدر والفوتر يظهران بشكل صحيح
- [ ] تسجيل الخروج يعمل من القائمة/الهيدر
